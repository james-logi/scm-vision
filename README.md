# 출고 재고 검수 관리 | AI 비전

> P-Box 단위로 처리되던 출고 데이터를 AI 비전으로 개별 단위까지 추적하여,
> 단위 변환 지점의 정보 손실을 차단하고 채찍효과의 미시적 시작점을 제거하는
> **출고 검수 시스템**

**인천대학교 동북아물류대학원 · SCM 캡스톤 프로젝트**
관리자: James Jeong · 발표일: 2026-06-16 (예정)

## 1. 문제 정의 (Problem Statement)

### 1.1 현장 시나리오
롯데웰푸드 빵 생산 공장에서 다음과 같은 운영 흐름이 존재한다.

```
[생산동]
  컨베이어에서 빵이 개별(EA) 단위로 생산됨
        ↓
  작업자가 P-Box(플라스틱 박스)에 적재
        ↓
[물류동]
  P-Box 단위로 재고 관리 / 출고 처리
        ↓
[매장 · 대리점]
  빵 개별(EA) 단위로 수령 · 판매
```

### 1.2 핵심 문제
- 물류동은 P-Box 단위로만 출고를 처리하지만, 매장은 개별(EA) 단위로 수령한다.
- 작업자가 P-Box에 빵을 적재할 때 발생하는 수량 누락·혼입은 **시스템적으로 추적되지 않는다.**
- 결과: 시스템 재고와 실제 출고량 사이의 불일치 → 매장 클레임 → 재발주 → 정보 왜곡 확산.

### 1.3 SCM 이론적 본질
본 문제는 단순한 검수 자동화 과제가 아니다. SCM 관점에서 다음 세 가지 학술적 개념의 교차점에 있다.

1. **Inventory Record Inaccuracy (IRI)**
   - 시스템 재고와 실물 재고의 불일치는 안전재고 산정의 기준값을 훼손한다.
   - DeHoratius & Raman (2008) 등은 IRI가 SCM 전체 성과를 좌우하는 숨겨진 변수임을 지적했다.

2. **Information Distortion at Unit Conversion**
   - 개별(EA) → 케이스(P-Box) 단위 변환 지점에서 정보가 사라진다.
   - 이는 단위 변환을 거치는 모든 공급망에서 발생하는 구조적 정보 손실이다.

3. **Bullwhip Effect의 미시적 시작점**
   - 출고 단계의 작은 누락 → 매장의 재발주 → 상위 단계의 수요 왜곡 → 채찍효과.
   - 채찍효과의 거시적 분석은 많지만, **시작점에서의 차단 메커니즘**은 거의 다뤄지지 않는다.

## 2. 솔루션 (Solution)

### 2.1 한 줄 정의
> AI 비전으로 P-Box 안의 빵 종류와 수량을 출고 직전에 자동 검수하고,
> Claude의 SCM 추론으로 패턴·원인·대응을 제시하는 시스템.

### 2.2 핵심 컨셉: Dual-Unit Tracking (이중 단위 추적)
기존 WMS는 P-Box 단위로만 재고를 관리한다. 본 시스템은 **P-Box 단위와 개별(EA) 단위를 동시에 추적**하여 단위 변환 지점의 정보 손실을 원천 차단한다.

### 2.3 4단계 워크플로우
```
[1] 이미지 업로드 → [2] AI 비전 검수 → [3] OK/NG 판정 + 수정
                                            ↓
                       [6] 출고 처리 ← [5] 재고 모니터링 ← [4] 데이터 반영
```

### 2.4 두 페르소나, 두 화면
- **작업자 모드**: 큰 ✅/⚠️ 표시, 큰 버튼, 즉시 행동 유도. 정보 최소화.
- **관리자 모드**: KPI · 트렌드 · 작업자 리스크 · Claude 추론. 정보 밀도 최대화.

## 4. 기술 스택

- **Frontend**: Streamlit + 커스텀 CSS (디자인 톤: 다크 네이비 + 무광 골드)
- **AI Vision**: Claude API (claude-opus-4-5) 멀티모달
- **AI Reasoning**: Claude API — 패턴 분석 + 자연어 리포트
- **Data**: Pandas + JSON (가벼운 in-memory + 파일 영속화)
- **Demo Safety**: Mock 모드 지원 (인터넷 장애·API 비용 0원)

### 4.1 기술 선택 논거: Vision LLM vs YOLO

본 시스템은 비전 검수 엔진으로 **Claude Vision (Vision LLM)** 을 채택했다.
객체 인식 분야의 대표 모델인 **YOLO**도 진지하게 고려했으나, 본 프로젝트의 요구사항에는 Vision LLM이 더 적합한 것으로 판단했다.

**선택 근거 3가지**

| # | 근거 | 설명 |
|---|------|------|
| 1 | **SKU 확장성** | 식품 제조 현장은 신제품 출시 주기가 짧다. YOLO는 매번 데이터 수집·재학습이 필요하지만, Vision LLM은 SKU 등록부에 시각적 특징을 자연어로 기술하면 즉시 인식 가능 |
| 2 | **의미적 추론** | YOLO는 좌표와 클래스만 반환. Vision LLM은 라벨 가독성·비닐 반사·유통기한 OCR을 단일 호출에서 함께 반환 |
| 3 | **개발 일정** | 30일 캡스톤에서 YOLO 학습(1~2주)에 시간을 쓰면 전체 시스템 통합이 밀린다. Vision LLM은 당일 사용 가능 |

**한 페이지 비교 요약**

| 항목 | Vision LLM (선택) | YOLO |
|------|-------------------|------|
| 학습 필요 | 불필요 (zero-shot) | 필수 (수백 장/SKU) |
| 개발 기간 | 당일 사용 가능 | 1~2주 학습 + GPU 자원 |
| 추론 속도 | 2~5초 (API) | 50~200ms (로컬 GPU) |
| 정확도 (본 데이터) | 92~98% | 95%+ (학습 후) |
| 새 SKU 대응 | 라벨 등록만으로 즉시 | 데이터 재수집 + 재학습 |
| 출력 형태 | 구조화 JSON + 자연어 추론 | 박스 좌표 + 클래스 라벨 |
| 부가 기능 | OCR · 라벨 평가 · 추론 통합 | 별도 파이프라인 필요 |
| 비용 구조 | API 호출당 과금 | 초기 GPU 투자 → 무료 |
| 인터넷 의존 | 필수 | 오프라인 가능 |
| 적합 시나리오 | 다품종 · 변경 잦은 환경 | 단일 품목 · 대량 처리 |

**한계 인정 + 하이브리드 확장 방향**

Vision LLM의 추론 속도(2~5초/이미지)와 API 비용은 명백한 약점이다. 대량 처리가 요구되는 컨베이어 풀스피드 환경에서는 두 기술의 강점을 결합한 **하이브리드 구조**가 더 적합하다.

```
[Hybrid Pipeline]
  ① YOLO       : 모든 P-Box를 빠르게 1차 스캔 (50ms/박스)
  ② 카운트 불일치 또는 confidence < 임계값 → NG 후보 플래그
  ③ Claude Vision : NG 후보만 정밀 분석 (혼입 · 라벨 · 자연어 권고)
  ④ WMS 실시간 통합 + 작업자에게 자연어 안내
```

본 프로토타입은 Vision LLM 단독 구조로 구현되었으며, 위 하이브리드 구조는 **후속 연구 과제**로 남긴다. 본 프로젝트의 핵심 기여는 비전 모델 자체가 아니라 **비전 검수 + WMS 실시간 통합 + Dual-Unit Tracking 아키텍처**의 검증이며, 비전 모델은 교체 가능한 컴포넌트로 설계되었다.

대화형 비교는 Streamlit 앱의 [5. 기술 선택 논거] 페이지에서 확인 가능하다.

## 6. 기대 효과

### 6.1 정량적 효과
- 클레임 발생률 감소 → 직접 비용 절감
- 매장 안전재고 최적화 가능 → 운영 자본 효율화
